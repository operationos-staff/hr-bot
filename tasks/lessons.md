## [2026-04-27] Supabase `.in()` с `null` не работает

**Контекст:** В `getRanking()` использовал `.in('qualified', [true, null])`, чтобы вытащить ✅ и 🟡 кандидатов одним запросом.

**Ошибка:** PostgREST превращает `null` в литерал-строку `"null"`, и Postgres возвращает `invalid input syntax for type boolean: "null"`. Запрос отдаёт пустой результат + ошибку в логах.

**Правило:** для фильтра «значение есть в списке ИЛИ null» используем `.or('col.is.null,col.eq.true')`. Альтернативы:
- `.is('col', null)` — только null;
- `.eq('col', true)` — только true;
- `.or('col.is.null,col.eq.true')` — оба.

Никогда не суём `null` в массив для `.in()`.
