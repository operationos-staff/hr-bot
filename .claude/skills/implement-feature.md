# Skill: implement-feature

Используй этот навык когда нужно реализовать новую фичу по спецификации.

## Workflow

1. **Прочитать** SPEC_TEMPLATE.md для понимания требуемого формата
2. **Создать** спецификацию фичи по шаблону или прочитать уже готовую
3. **Проверить** TECH_SPEC.md на зависимости от других модулей
4. **Обновить БД** (если нужны новые поля): делегировать database-architect
5. **Реализовать** код: делегировать backend-engineer или habr-parser/hh-api-client
6. **Ревью**: делегировать qa-reviewer
7. **Обновить** TECH_SPEC.md если добавились новые поля или эндпоинты
8. **Проверить** что CLAUDE.md не нужно обновлять (только если изменилась архитектура)

## Чеклист фичи
- [ ] Спецификация написана по шаблону (все 6 блоков)
- [ ] Миграция создана если нужны новые поля
- [ ] Код соответствует правилам (ESM, logger, try/catch)
- [ ] qa-reviewer ревью пройдено
- [ ] TECH_SPEC.md обновлён

## Не делать
- Не начинать код без спецификации
- Не добавлять зависимости без проверки package.json
- Не менять логику фильтрации без явного запроса
